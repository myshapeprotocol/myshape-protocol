# CPS-0001 — Continuity Protocol Core

**v1.0-RC** · Apache 2.0 · [The Continuity Lab](https://thecontinuitylab.org)

> A Continuity Protocol does not standardize how continuity is measured.
> It standardizes how continuity assertions are **represented, exchanged, and verified**.

---

## What Is This?

CPS-0001 defines the **Continuity Receipt** — a cryptographically verifiable statement that an observer collected sufficient evidence supporting the continuity of a subject over a bounded interval of time.

It is **engine-independent**. No sensor type, algorithm, or hardware requirement appears in the specification. A third party can build a completely different evidence engine — using different sensors, different algorithms, different hardware — and produce a valid Continuity Receipt that any conforming verifier can process.

## Why Does It Exist?

AI can now generate faces, voices, and behavior. "Who are you?" stops being the right question.

The right question: **who continues to be you?**

Continuity verification answers: is the subject at time t₁ the same subject observed at time t₀ — across the entire interval between them?

This is **not identity verification**. It doesn't answer "who is this?" It answers "is this the same subject as before?" The distinction is fundamental.

## Quickstart

```bash
npm install @thecontinuitylab/myshape
```

```typescript
import { buildReceipt, verifyReceipt } from "@thecontinuitylab/myshape";

// An evidence engine produces evidence...
const receipt = buildReceipt({
  evidence: [{ engineId: "EE-001", engineVersion: "1.2.0", confidence: 0.85, payload: { entropy: 0.72 }, payloadDigest: "sha256:a1b2..." }],
  interval: { start: "2026-07-22T10:00:00.000Z", end: "2026-07-22T10:00:08.000Z", coverageMs: 8000 },
  subject: { id: "sha256:abcd...", type: "embodied" },
  issuer: { id: "sha256:issuer...", publicKey: "..." },
});

// Any conforming verifier can process it — no MyShape engine required
const result = await verifyReceipt(receipt);
// { status: "VALID" }
```

## Protocol at a Glance

```
ContinuityReceipt
├── Assertion   — what is claimed (3 closed assertions)
├── Evidence    — why it is believed (opaque engine payloads)
├── Context     — when / where / subject
├── Composability — predecessor reference (chain)
└── Signature   — who claims it
```

### What It Asserts

| # | Assertion |
|:---|:---|
| A₁ | **Observation Occurred** — real sensor data was collected |
| A₂ | **Continuity Maintained** — same subject at t₀ and t₁ |
| A₃ | **Receipt Integrity** — not modified since issuance |

### What It Does NOT Assert

Identity · Humanity · Liveness · Uniqueness (Sybil Resistance) · Location · Intent

### V₁–V₇ Verification Contract

| Step | Check | Engine-Dependent? |
|:---|:---|:---|
| V₁ | Schema validity | No |
| V₂ | Signature validity | No |
| V₃ | Assertion consistency | No |
| V₄ | Temporal consistency | No |
| V₅ | Evidence reference integrity | No |
| V₆ | Freshness | No |
| V₇ | Predecessor reference | No |

**None of the seven checks require knowing which evidence engine was used.**

## Protocol Success Criteria (§7.5)

A conforming implementation should answer **yes** to all three:

| # | Question | Test |
|:---|:---|:---|
| Q₁ | Can another team implement without using MyShape? | Give them only this spec. Can they produce a valid receipt? |
| Q₂ | Can two different engines produce interoperable receipts? | Run EE-001 and a third-party engine. Do both pass V₁–V₇ in the same verifier? |
| Q₃ | Can an app consume a receipt without knowing how evidence was generated? | Build an app that accepts a receipt and runs V₁–V₆ — without importing MyShape engine code. |

## Architecture

```
                Applications
                     |
              CPS-0001 Receipt
                     |
           Verification Contract (V₁–V₇)
                     |
        -------------------------
        |                       |
 Evidence Producers        Issuers
        |
 ----------------
 |              |
MyShape      Others
Engine       Engine
```

The protocol standardizes the **object**, not the engine. Evidence engines are implementation details. A third-party engine that produces a conforming receipt is a success, not a threat.
